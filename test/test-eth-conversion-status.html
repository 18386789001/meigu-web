ETH转换状态显示测试

测试场景说明

当API返回的 failure_msg 为 "当前币种为eth直接完成订单" 时，前端应该将充值订单状态显示为成功，并显示 "ETH已兑换为USDT" 的提示信息。

测试用例

测试用例1: 正常成功状态

✓ 成功
API返回: status = 1, failure_msg = null
预期显示: 绿色圆点 + "成功"

测试用例2: ETH转换成功状态

✓ ETH已兑换为USDT
✓ 成功
API返回: status = 0, failure_msg = "当前币种为eth直接完成订单"
预期显示: 绿色圆点 + "ETH已兑换为USDT" + "成功"

测试用例3: 正常确认中状态

⏳ 确定中
API返回: status = 0, failure_msg = null 或其他值
预期显示: 黄色圆点 + "确定中"

测试用例4: 正常失败状态

✗ 失败
API返回: status = 2, failure_msg = "其他错误信息"
预期显示: 红色圆点 + "失败"

测试用例5: ETH转换失败状态（不应出现）

✗ 失败
API返回: status = 2, failure_msg = "当前币种为eth直接完成订单"
预期显示: 红色圆点 + "失败"（这种情况理论上不应该出现）

API接口测试

接口调用

接口地址: https://jpmx.xyz/api/rechargeBlockchain/list
请求方法: GET
请求参数: page_no=1&language=zh-CN
预期返回: 包含 failure_msg 字段的充值记录列表

实现要点

  • ✅ 修改了充值历史组件的状态显示逻辑
  • ✅ 添加了对特定 failure_msg 的检测
  • ✅ 当 failure_msg 为 "当前币种为eth直接完成订单" 时显示成功状态
  • ✅ 添加了多语言支持（中文、英文、日文、韩文、德文、法文）
  • ✅ 更新了API调用，添加了language参数
  • ✅ 保持了原有的其他状态显示逻辑不变
