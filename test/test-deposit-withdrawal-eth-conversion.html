DepositAndWithdrawal页面ETH转换状态测试

API接口信息

接口地址: https://jpmx.xyz/api/rechargeBlockchain/list?page_no=1&language=zh-CN

请求方法: GET

请求参数: page_no=1&language=zh-CN

当前状态: 需要登录认证（返回403错误）

修改内容说明

在 wap-vue/src/views/Record/DepositAndWithdrawal.vue 页面中，当API返回的 failure_msg 为 "当前币种为eth直接完成订单" 时，前端会将充值订单状态显示为成功，并显示 "ETH已兑换为USDT" 的提示信息。

修改的具体内容

1. 模板修改

<div class="status flex"> <template v-if="getStatusValue(item) == 0 && !isEthConverted(item)"> <div class="status-icon status-icon-color3 mr-2"></div> {{ $t('confirming') }} </template> <template v-if="getStatusValue(item) == 1 || isEthConverted(item)"> <div class="status-icon status-icon-color1 mr-2"></div> <template v-if="isEthConverted(item)"> {{ $t('ethConvertedToUsdtNotice') }} </template> <template v-else> {{ $t('success') }} </template> </template> <template v-if="getStatusValue(item) == 2 && !isEthConverted(item)"> <div class="status-icon status-icon-color2 mr-2"></div> {{ $t('fail') }} </template> </div>

2. 新增函数

// 判断是否为ETH转换状态 const isEthConverted = (item) => { if (selectIndexActive.value == 0) { // 银行卡充提记录不涉及ETH转换 return false } else { // 数字货币充提记录，检查failure_msg return item.failure_msg === '当前币种为eth直接完成订单' } }

状态显示逻辑

状态判断逻辑

确认中状态: status == 0 && failure_msg != "当前币种为eth直接完成订单"
成功状态: status == 1 || failure_msg == "当前币种为eth直接完成订单"
失败状态: status == 2 && failure_msg != "当前币种为eth直接完成订单"
ETH转换显示: failure_msg == "当前币种为eth直接完成订单" 时显示 "ETH已兑换为USDT"
